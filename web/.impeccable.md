# Design Context

## Target audience
Polish-first (English as a toggle) software developers, mostly intermediate Python
engineers, learning Pydantic AI (an LLM agent framework) and the wider Pydantic
ecosystem. They read carefully, run real code, and value precision and depth. The
quality bar they expect is react.dev/learn and Microsoft Learn.

## Use cases
- Work through the tutorial chapter by chapter, in Polish or English.
- Read rich explanations with diagrams, execution graphs, tables, and runnable examples.
- Run companion Python projects from `/sandbox`.
- Take an end-of-chapter quiz; track score and progress across the whole course.
- Use the site as a reference, jumping to specific chapters and linking out to source docs.

## Brand personality / tone
Authoritative but warm. An editorial technical manual: a beautifully typeset interactive
textbook crossed with a tech magazine. Confident and precise, occasionally witty (the
odd meme is welcome). Never corporate-bland, never neon-cyberpunk AI slop.

# Aesthetic Direction: "Editorial Technical Manual"

A printed-textbook feel rendered for the screen. Generous reading measure, strong
typographic hierarchy, warm paper and ink, a single confident accent.

## Typography
- Display / headings: **Fraunces** (variable serif, optical sizing) - editorial, distinctive.
- Body / UI: **Hanken Grotesk** - clean, highly readable grotesque (not Inter/Roboto).
- Code / mono: **JetBrains Mono** - used only where code legitimately lives.
- Fluid modular scale with clamp(). Vary weight and size hard for hierarchy.

## Color (OKLCH, tinted neutrals)
- Accent ("Pydantic red"): warm crimson-pink, roughly oklch(0.58 0.20 18).
- Secondary highlight: warm amber, roughly oklch(0.80 0.13 75), used sparingly.
- Neutrals tinted toward the red hue (warm grays), never pure gray.
- Light ("paper"): warm near-white background (white is acceptable), warm ink text.
- Dark ("ink"): warm charcoal background (never pure black), warm paper text.
- No cyan-on-dark, no purple-to-blue gradients, no gradient text, no glow-by-default.

## Layout & space
- Asymmetric, editorial. Wide reading column with a margin rail for notes/footnotes.
- Varied rhythm, not uniform card grids. Flatten hierarchy; avoid cards-in-cards.
- Collapsible sidebar for chapter nav, content area with a right-hand "on this page" rail.

## Motion
- One orchestrated, staggered page-load reveal per page (animate-ui).
- Transform/opacity only, exponential ease-out, respect reduced-motion.
- No bounce/elastic. Motion communicates state, it does not decorate.

## Signature / differentiation
Per-chapter animated **execution graphs** (how an agent run flows) and the warm
paper-and-ink editorial type system. Someone should ask "how was this made", not
"which AI made this".
